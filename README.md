# CropXpert AI 🌾

**An Intelligent AI-Powered Agricultural Decision Support Platform**

CropXpert AI helps farmers make data-driven decisions by recommending the best
crops for their land using machine learning, live weather data, automated PDF
reports, and a full recommendation history.

---

## Features

| Feature | Description |
|---------|-------------|
| AI Crop Recommendation | RandomForest model returns Top-3 crops with confidence scores |
| Weather Automation | Enter a city — climate is detected automatically via OpenWeatherMap |
| Recommendation History | Every query is saved to MySQL and viewable as a history list |
| PDF Report | Download a formatted report with farmer info, weather, and recommendations |
| User Auth | Signup / Login with bcrypt-hashed passwords |

## Setup — Step by Step

### 1. Clone the repository

```cmd
git clone https://github.com/your-username/cropxpert.git
cd cropxpert
```

### 2. Install Python dependencies

```cmd
pip install flask flask-cors mysql-connector-python pandas werkzeug python-dotenv scikit-learn fpdf2 requests gunicorn numpy
```

Or using the requirements file:

```cmd
pip install -r requirements.txt
```

### 3. Set up MySQL database

Open MySQL Workbench or the MySQL command line and run:

```sql
CREATE DATABASE cropxpert;
USE cropxpert;

CREATE TABLE users (
    id       INT AUTO_INCREMENT PRIMARY KEY,
    name     VARCHAR(100),
    phone    VARCHAR(20),
    email    VARCHAR(100) UNIQUE,
    gender   VARCHAR(10),
    city     VARCHAR(100),
    password VARCHAR(255)
);
```

Then run the full upgrade script for history tables:

```cmd
mysql -u root -p cropxpert < backend/db_upgrade.sql
```

### 4. Configure environment variables

```cmd
copy backend\.env.example backend\.env
```

Open `backend/.env` and fill in:

```
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=cropxpert
DB_PORT=3306
OPENWEATHER_API_KEY=your_key_from_openweathermap
```

### 5. Train the ML model

```cmd
cd backend
python train.py
cd ..
```

This creates `backend/model/crop_model.pkl` and `backend/model/encoders.pkl`.

### 6. Start the Flask backend

```cmd
cd backend
python app.py
```

You should see:

```
[DB] Pool created.
[Dataset] 512 rows loaded.
[ML] Model loaded — 496 classes.
 * Running on http://0.0.0.0:5000
```

### 7. Open the frontend

Open `homepage.html` directly in your browser, or use VS Code Live Server.

With Live Server: right-click `homepage.html` → Open with Live Server
→ opens at http://127.0.0.1:5500/homepage.html

## Getting an OpenWeatherMap API Key

1. Go to https://openweathermap.org/
2. Click Sign Up — it is free
3. Verify your email
4. Go to your profile → API Keys
5. Copy the default key
6. Paste it into `backend/.env` as `OPENWEATHER_API_KEY=`

Note: New keys take up to 2 hours to activate after registration.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `[DB] Pool failed` | Check MySQL is running and .env password is correct |
| `[ML] File not found` | Run `python train.py` inside the backend folder first |
| `City not found` | Check spelling or try a nearby larger city |
| `Invalid API key` | Wait 2 hours after creating the OpenWeatherMap key |
| CORS error in browser | Make sure Flask is running on port 5000 |
| `ModuleNotFoundError` | Run `pip install -r requirements.txt` again |
